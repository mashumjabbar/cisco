Introduction
Note
In this activity, you will interact with a recording of a lab, made on actual devices, rather than with the actual devices. In this activity, when entering a command, the entire command must be entered exactly as stated, including matching the case and spaces; shortcuts and tab completion are not supported. In GUIs, you will only be able to use the menu items required for the activity, you will not be able to use other menu items.
To authenticate the wireless client that needs to join the WLAN that you have created in the previous task, you must define at least one RADIUS server to be used for authentication by your WLC. In this task, you will define a RADIUS server.

Task1: Define a RADIUS Server

  • Procedure1:
    You will begin at the Monitor screen. Click the Security tab.
    You will see the Security General screen. In the menu bar on the left, you will see that the AAA option has been expanded.
  • Procedure2:
    Look down the list at the RADIUS options. You will see the Authentication, Accounting, Fallback, DNS, and Downloaded AVP. Click the Authentication option.
  • Procedure3:
    The next screen is the RADIUS Authentication Servers. As you can see, there are four options here, but for this configuration you will leave all the options in their default state. You do need to add a new RADIUS server. To do that, go to the upper right side of the screen and click New.
  • Procedure4:
    Now you will see the RADIUS Authentication Servers > New screen. You will leave all these options at their default settings. You do need to add an IPv4 address for the RADIUS server and a shared secret password. In the IP Address dialog box, enter 172.31.1.6 and press Enter. Leave the Shared Secret set to ASCII.
    Note
    This is a simulation. In a real environment, you would not need to press Enter and submit every entry. You would first fill in all the information on the screen and only submit it in the end, using the Apply button.
  • Procedure5:
    In the Shared Secret dialog box, enter iforgot and press Enter.
    Note
    No characters are displayed when you enter the Shared Secret and Confirm Shared Secret entries. These passwords are case sensitive. Type carefully.
  • Procedure6:
    In the Confirm Shared Secret dialog box, enter iforgot and press Enter.
  • Procedure7:
    If everything looks good, go to the upper right side of the screen and click Apply.
    Your configuration should look like the following:
    When you click Apply, you will be taken back to the RADIUS Authentication Servers Page.
  • Procedure8:
    On this screen, you should see the new server that you have just entered. Validate some of the more important values you configured when you created the new RADIUS server. Specifically, check that the Network User and Management are both enabled. Since you only have one RADIUS server, the Server Index should be set to 1. Verify, that the IPv4 address is 172.31.1.6 and that the port used for communication is 1812.
    Click Next to move to the right side of the screen. In non-simulation environment, you would use the scrollbar.
    IPSec is disabled and the Admin Status is enabled.
  • Procedure9:
    Finally, go to upper-right side of the screen and click Save Configuration.
  • Procedure10:
    In the popup window, click OK.
  • Procedure11:
    You have completed this activity when you have attained this result:
    • You have added a RADIUS server on the WLC and seen it on the RADIUS Authentication Servers page.

  • Defining a TACACS+ Server
    In the Security menu, you can very similarly define TACACS+ servers. However, there is one main difference—you can configure a RADIUS server to authenticate management as well as wireless users, while you can define TACACS+ servers only to control access to the WLC itself.
    You would select TACACS+ server over RADIUS, when requiring full AAA functionality for administrative user access. When requiring authorization, permissions are mapped to various tabs that correspond to the seven menu bar items in the GUI.
