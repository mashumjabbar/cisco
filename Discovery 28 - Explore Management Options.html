Introduction
Note
In this activity, you will interact with a recording of a lab, made on actual devices, rather than with the actual devices. In this activity, when entering a command, the entire command must be entered exactly as stated, including matching the case and spaces; shortcuts and tab completion are not supported. In GUIs, you will only be able to use the menu items required for the activity, you will not be able to use other menu items.
Up to this point, you have been using the GUI interface to manually configure and verify the configuration of the WLC. This works very well if you only have a few WLCs. What if you have 25 or 50 WLCs spread across your organization? The configuration can be a little daunting, but managing them can be a challenge, especially if the WLCs are spread across many geographically dispersed locations. A simple way to manage a network of this scope is to use the SNMP and a centralized management console. In this activity, you will configure the SNMP location for the WLC and determine the events that will generate SNMP traps.
Note
This is the seventh discovery in the series of the Cisco WLC discoveries. This discovery starts where the previous discovery has ended. The topology shows the relevant devices and addresses configured in this series of related discoveries.

  • Procedure1:
    Begin at the Monitor Summary screen and click the Management tab in the menu bar at the top of the screen.
    You will then see the Management Summary screen.
  • Procedure2:
    To begin configuring SNMP, go to the left menu bar, and click the arrow next to SNMP to expand the SNMP options.
  • Procedure3:
    Next click the General menu option under SNMP in the menu options listed on the left-side of the screen.
    You will see the following screen. Notice the name. The WLC has a name CCNA-Core, which has been preconfigured during the initial setup of the WLC. You will change the name to RCDN_Lab-Pod1.
  • Procedure4:
    Place the mouse cursor at the end of CCNA-Core in the Name dialog box. Now, use backspace to remove it. Then type RCDN_Lab-Pod1 in the Name dialog box and press Enter.
    Note
    This is a simulation. In a real environment, you would not need to press Enter and submit every entry. You would first fill in all the information on the screen and only submit it in the end, using the Apply button.
  • Procedure5:
    Next, type RCDN Building 5 in the Location dialog box and press Enter.
  • Procedure6:
    Finally, in the Contact dialog box, type Bill Smith and press Enter.
    Here is the view of the correct configuration.
  • Procedure7:
    If your configuration looks good, go to upper-right side of the screen, and click Apply.
  • Procedure8:
    Then click Save Configuration. You will need to click OK when the dialog box appears after you click Save Configuration. The screen will not change. This is the new SNMP System Summary.
  • Procedure9:
    Next you should look at the SNMP Trap Controls. In the Management section on the left side of the screen, choose the Trap Controls option under SNMP.
    This is the first screen that you will see. Notice that there are three options in the General tab and they are all checked. On the right side of the screen, there is a check box that, when used, will select all the options in a tab. You will see this check box on each subsequent screen, and it makes selecting all the options easier. Be careful, because if you select all the trap options, you will receive a rather large number of SNMP trap notifications from the WLC. Using the default settings is a good place to start, and then adjust the traps to meet your needs.
  • Procedure10:
    Click the Client tab to view the Client SNMP trap options.
  • Procedure11:
    Click the AP tab to view the AP SNMP trap options.
    Here are the SNMP traps that will be collected for APs.
  • Procedure12:
    Click the Security tab to view the Security SNMP trap options.
    Here are the Security SNMP options.
  • Procedure13:
    Next click the Auto RF tab to view the Auto RF SNMP trap options.
    Here are the Auto RF Trap Options.
  • Procedure14:
    Next click the Mesh tab to view the Mesh SNMP trap options.
  • Procedure15:
    Next go to the menu bar on the left and click the arrow next to Tech Support.
  • Procedure16:
    Click System Resource Information to view system resource information.
    The information is divided into four segments: Current CPU(s) Usage, Individual CPU Usage, System Buffers, and Web Server Buffers. Because you are in a lab environment, system resources are lightly used. Now you have an idea of the types of resource information that can be gathered if the performance of the WLC appears slow.
  • Procedure17:
    Next click the arrow next to Logs.
    As you can see, there are two options: Config and Message logs. The Config option allows you to enter an IP address to which to send the syslog messages to. Up to three syslog servers can be added to the Cisco WLC.
  • Procedure18:
    Next click Message logs.
    As you can see, there are several messages in the log file, and the WLC is not particularly busy at this point. If you had many APs and clients connected to the WLC, the number of messages in the log file could be voluminous, and you may want to send the message logs to a centralized server so they can be parsed more easily when trying to resolve a problem.
  • Procedure19:
    You have completed this activity when you have attained these results:
    • You have navigated to the Management option and viewed the Summary screen output.
    • You have used the left menu bar to navigate to SNMP Summary screen where you changed the name of the SNMP system from CCNA-Core to RCDN_Lab-Pod1. You also added the location and the contact name.
    • You have opened the Trap Controls option and viewed the available information in each sub-tab.
    • You have opened the Tech Support menu on the left menu bar and viewed the information available in the System Resource Information screen.
    • You have viewed the options in the Logs menu.
