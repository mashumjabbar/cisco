Introduction
Note
In this activity, you will interact with a recording of a lab, made on actual devices, rather than with the actual devices. In this activity, when entering a command, the entire command must be entered exactly as stated, including matching the case and spaces; shortcuts and tab completion are not supported. In GUIs, you will only be able to use the menu items required for the activity, you will not be able to use other menu items.
In this activity, you will configure a DHCP scope to provide IPv4 addresses to the hosts that will be on VLAN 10. As you know, DHCP can be deployed from many different platforms. It is possible to deploy DHCP from a dedicated server, a Windows server, a switch, or from a router. In this case, you will be deploying the DHCP scope from the WLC.
In the previous discovery, you created a new VLAN 10 and the IPv4 address of the interface of 10.10.10.1/24. Using VLAN 10, you will later create a WLAN. As the clients will be joining the WLAN, you will want to have a method of providing IPv4 addresses in a specific range and avoid using IPv4 addresses that have already been deployed. You can set this up in advance, configuring a DHCP scope.
Note
This is the fourth discovery in the series of the Cisco WLC discoveries. This discovery starts where the previous discovery has ended. The topology shows the relevant devices and addresses configured in this series of related discoveries.

  • Procedure1:
    To begin the process of configuring a DHCP scope in the WLC, you will start on the Monitor Summary screen. Click the Controller option in the top menu bar. The screen that you will initially see is the Controller General screen. You saw this screen when you began to configure an interface in the WLC in the previous activity.
  • Procedure2:
    Now go to the menu bar on the left side of the screen and click the triangle next to the Internal DHCP Server option.
  • Procedure3:
    You will see two options: DHCP Scope and DHCP Allocated Leases. Click DHCP Scope.
    When the screen opens, you see the DHCP Scopes that are configured on the WLC. As you can see, one scope has been configured already. This scope is the day0-dhcp-mgmt scope. This scope is used with Service Port during initial configuration. This scope is not part of this set of activities.
  • Procedure4:
    To add a new DHCP Scope, click Next to move to the right of the screen and then click New in the upper right portion of the screen.
  • Procedure5:
    A new screen will be displayed. The dialog box provides a space for you to add the name of the new DHCP Scope. In the dialog box, enter VLAN-10 and press Enter.
    Note
    This is a simulation. In a real environment, you would not need to press Enter and submit every entry. You would first fill in all the information on the screen and only submit it in the end, using the Apply button.
  • Procedure6:
    Here is a view of the new scope with VLAN-10 added. Click Apply in the upper right corner of this screen.
  • Procedure7:
    When you click Apply, you will be taken back to the DHCP Scopes screen. On this screen, you can see that VLAN-10 has an IPv4 address pool of 0.0.0.0 â€“ 0.0.0.0 (further to the right you would see that the lease time is 1 day, and the status is disabled). To edit these settings and enable the scope, click VLAN-10.
    The next screen is DHCP Scope > Edit. You can see that the DHCP Scope name is what you entered on the previous screen (VLAN-10).
  • Procedure8:
    Now, you need to begin the process of completing the DHCP Pool for VLAN-10. In the Pool Start Address dialog box, enter 10.10.10.100 and press Enter.
  • Procedure9:
    In the Pool End Address dialog box, enter 10.10.10.200 and press Enter.
  • Procedure10:
    In the Network dialog box, enter 10.10.10.0 and press Enter.
  • Procedure11:
    In the Netmask dialog box, enter 255.255.255.0 and press Enter.
  • Procedure12:
    In the Default Router dialog box, you can add up to three different default routers. In this case, you only need to add one. In the first Default Router dialog box, enter 10.10.10.253 and press Enter.
  • Procedure13:
    In the DNS Server dialog box, enter 9.9.9.9 and press Enter.
  • Procedure14:
    Finally, there is a drop-down menu next to Status. Change the Status to Enabled. Many times, users forget to enable their DHCP Scope and wonder why it is not working properly when all the other parameters are configured properly.
  • Procedure15:
    Click Apply.
  • Procedure16:
    Click Save Configuration to commit your changes.
  • Procedure17:
    Click OK in the pop-up window that appears.
    The screen will now display the DHCP Scopes screen. You should see the VLAN-10 and day0-dhcp-mgmt scopes.
  • Procedure18:
    Verify that the information you configured for VLAN-10 is correct and the Status is Enabled.
    Normally you would use the scroll bar on the bottom of the screen to examine the fields on the far right; in this simulation you will use the Next button to see this information.
  • Procedure19:
    You have completed this activity when you have attained these results:
    • You have navigated to the Controller screen and selected Internal DHCP Server.
    • You have examined the two options from the drop-down menu under Internal DHCP Server.
    • You have added a new DHCP server scope for VLAN-10.
    • You have configured the starting and ending address pools, the network, subnet mask, default router and enabled the scope.
    • You have verified the scope has been configured properly.
