The customer has requested CCS to configure a system logging (syslog) server to log all the messages. The FileServer is configured as a syslog server. Therefore, you must configure the Branch router and SW1 switch to forward the log messages to the FileServer.
  • Configure logging on the Branch router to forward all messages that have severity levels up to 4 to the syslog server.
  • Configure logging on the SW1 switch to forward all messages that have severity levels up to 5 to the syslog server.

Note
If you shut down an interface on a real router or switch, the connected device will see it as "down/down." Due to virtualization specifics, Cisco IOL (Cisco IOS Software on Linux) behavior is slightly different. If you shut down an interface on a router or switch, the connected device will see it as "up/up." In Cisco IOL, the status of an interface can only be "up/up" or "administratively down/down." Also, in the virtual lab environment, all interfaces are Ethernet interfaces and not FastEthernet or GigabitEthernet interfaces, which you are likely to encounter in networks today.

Configuration Tips
Configuration Tips
  • Use the logging ip address command to configure the IP address of the host that will receive the system logging (syslog) messages.
  • Use the logging trap level command to limit messages that are logged to the syslog servers based on severity. Use the logging trap command in the global configuration mode. You can use numbers or the word for the level in the logging trap command.
  • Use the show logging command to display the state of system logging (syslog) and the contents of the standard system logging buffer. Use the show logging command in privileged EXEC mode.
  • Use the show running-config | include logging command to display the information about the configured logging host and the configured severity of the messages that are logged.

Answer Key

    Branch> enable Branch# configure terminal Enter configuration commands, one per line. End with CNTL/Z. Branch(config)# logging host 172.16.130.6 Branch(config)# logging trap warnings
  • Configure logging on the SW1 switch to forward all messages that have severity levels up to 5 to the syslog server.
    SW1(config)# logging host 172.16.130.6 SW1(config)# logging trap notifications

Questions
Questions
Question 1
Which command do you use to check syslog messages on the router?
The correct answer is "show logging." To display the state of system logging (syslog) and the contents of the standard system logging buffer, use the show logging command in privileged EXEC mode. Lastly, other show commands are not valid commands in Cisco IOS.
Question 2
What is the expected result, when you issue the show running-config | include logging command on SW1?
The correct answer is "logging trap notifications, logging host 172.16.130.6., logging synchronous" The logging host ip-address command identifies a syslog server that will receive logging messages. In this case, the syslog server resides on 172.16.130.6 IP address. The logging trap severity command limits the syslog messages that are sent to the syslog server. For example, if severity level 5 is configured, all messages with severity levels up to 5 will be displayed (Emergency, Alert, Critical, Error, Warning, and Notification).
Question 3
What is the expected result, when you issue the show running-config | include logging command on the Branch router?
The correct answer is "logging trap warnings, logging host 172.16.130.6., logging synchronous" The logging host ip-address command identifies a syslog server that will receive logging messages. In this case, the syslog server resides on 172.16.130.6 IP address. The logging trap severity command limits the syslog messages that are sent to the syslog server. For example, if severity level 4 is configured, all messages with severity levels up to 4 will be displayed (Emergency, Alert, Critical, Error, and Warning).
Question 4
You have received the following syslog message on the router:
*Apr 19 06:12:11.393: %LINK-3-UPDOWN: Interface Ethernet0/0, changed state to up
Which severity level is used for this message?
The correct answer is "errors." In this case, the syslog message indicates that the link layer of interface Ethernet0/0 went from down state to up. It is categorized with severity level 3, because in some scenarios "%LINK-3-UPDOWN" events can prevent the normal function of the switch on the network. For example, flapping interfaces could introduce various problems in your network. The severity levels go from 0 to 7. From most severe to least severe, the levels are Emergency, Alert, Critical, Error, Warning, Notification, Informational, and Debugging.
