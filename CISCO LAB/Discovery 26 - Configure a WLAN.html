Introduction
Note
In this activity, you will interact with a recording of a lab, made on actual devices, rather than with the actual devices. In this activity, when entering a command, the entire command must be entered exactly as stated, including matching the case and spaces; shortcuts and tab completion are not supported. In GUIs, you will only be able to use the menu items required for the activity, you will not be able to use other menu items.
This activity is relatively short but very important. You will perform the initial configuration for a WLAN. In a corporate environment, you may have a large number or small number of WLANs, depending on your needs. When you connect to a wireless network, you use an SSID. Each SSID is associated to a WLAN. For example, you may have a Guest SSID that only provides the internet access to guest users. You may have an Employee SSID that can provide differentiated levels of services to different users or groups of users based on their login credentials. In some situations, you may have a department WLAN for just the users in that department. Other users would be prevented from joining that WLAN by their user credentials. In previous activities, you created an interface in the WLC. Here you will create the WLAN that uses that VLAN.

Task1: Configure a WLAN

  • Procedure1:
    You are back the Monitor Summary Screen. The first step is to choose the WLANs option in the top menu bar.
  • Procedure2:
    You will see this screen next. As you can see, there are no WLANs configured on the WLC.
    To add a new WLAN to the WLC, go to the right and look for the dialog box that says, "Create New." Click the Go button next to the dialog box.
    A new screen will appear that prompts you to type in the profile name and SSID details.
  • Procedure3:
    There are four fields that you can configure but you will only be using three of them in this discovery. Look at the Type dialog box. It says WLAN. Click the down arrow. You will see that there are three options, Guest LAN, WLAN, and Remote LAN. Guest LAN is meant for users that come into your facility and need only the internet access. The Remote LAN would be for those users that are using Office Extend AP (OEAP). This option would allow remote uses to connect to the corporate network and have the same access as if they were at their desk. For example, a mobile user at a home office. Choose WLAN from the drop-down menu
  • Procedure4:
    The next option is Profile Name. Enter a descriptive name for the WLAN. Capitalization and spaces are permitted. Enter 1xauth1 in the Profile Name dialog box, and press Enter.
    Note
    This is a simulation. In a real environment, you would not need to press Enter and submit every entry. You would first fill in all the information on the screen and only submit it in the end, using the Apply button.
  • Procedure5:
    Now go to the SSID dialog box. Typically, the Profile Name and the SSID are the same. You could use different values, but it might cause confusion with your user community and that is something you really want to avoid. In the SSID Dialog box, enter 1xauth1 and press Enter.
  • Procedure6:
    The last option is ID. This option is the VLAN you want to associate with the WLAN. As you can see, the default is 1. Click the down arrow, and you will see that the option range is from 1 to 512. Choose 10 as this is the ID of the VLAN you have created earlier.
  • Procedure7:
    Remember to click Apply.
  • Procedure8:
    Click Save Configuration to commit the changes.
  • Procedure9:
    Upon receiving the pop-up window, confirm that you want to save the configuration by clicking OK.
    You will see this new screen. Notice at the top of the screen the heading is WLANs > Edit '1xauth1.' Verify that the information that you entered in the previous step have been configured properly.
    You also see that there are five tabs below the heading. They are General, Security, QoS, Policy-Mapping, and Advanced. In this activity, you will only use the General tab. As you can see, the General tab is the opening tab you see. The profile name, Type, and SSID have been carried forward from the initial WLAN configuration screen.
  • Procedure10:
    The most important configuration change you need to make is the Status option. Check the checkbox next to the Enabled option to change the status to Enabled.
    As you can see, there is a box next to Enabled. You need to click in the box next to Enabled to enable this WLAN. If you fail to click in the box next to Enabled, the WLAN will not become active. It often occurs that you made all the changes to the WLAN for Security and QoS, and when you go to try to join the WLAN, it is not there. You can spend a few minutes looking at your PC and WLC to ensure that they are working properly, only to discover that the Enabled box was not checked.
  • Procedure11:
    Click Apply.
  • Procedure12:
    Choose Save Configuration to apply the changes to the WLC.
  • Procedure13:
    Click OK in the pop-up window to confirm the changes.
  • Procedure14:
    Click WLANs in the left menu bar.
    You will see this screen. Use this screen to verify all the changes you have entered for WLAN are correct. Pay particular attention to the Admin Status. Ensure it is Enabled otherwise the WLAN will not be active.
    Click the Next button to see the rest of the screen. In a non-simulation environment, you would use the scrollbar.
    Note
    You can also remove a WLAN that you may no longer need or if it has been misconfigured by using the Remove option (from the blue drop-down arrow).
  • Procedure15:
    You have completed this activity when you have attained these results:
    • You have created a new WLAN.
    • You have added a Profile Name, selected a Type, added an SSID, and selected a VLAN ID.
    • You have verified the Profile Name and SSID and Enabled the WLAN.

  • Configuring Additional WLAN Parameters
    Beside VLAN assignment, Profile name and SSID, you can configure additional parameters for the WLAN:
    • Security: You can define user authentication and traffic encryption types.
    • QoS: QoS provides priority, including dedicated bandwidth. You can choose one of four different QoS levels per WLAN:
      • Platinum/Voice—ensures a high quality of service for voice over wireless.
      • Gold/Video—supports high-quality video applications.
      • Silver/Best Effort—supports normal bandwidth for clients. This level is the default setting.
      • Bronze/Background—provides the lowest bandwidth, e.g. for guest services.
    • Advanced: Provides you options to configure advanced performance and security features. Some of them include:
      • Enabling AAA override—RADIUS server can dynamically provide VLAN, ACL, and QoS assignment for the client.
      • Enabling the client session timeout.
      • Providing the maximum number of clients that are allowed per WLAN.
      • Enabling client exclusion.
      • Configuring peer-to-peer blocking.
      • Enabling band select feature, that tries to force clients to use 5GHz if possible.
      • Enabling load balancing feature, that tries to balance the clients between different APs. This is usually applicable for high density environments, when APs are installed close together.
      • Configuring WLAN as FlexConnect WLAN, which can then be switched locally instead of centrally on the WLC.
